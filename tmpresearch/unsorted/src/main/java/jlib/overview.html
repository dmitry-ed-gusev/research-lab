Библиотека системных и прикладных классов JLIB 4.9.9 (08.08.2011). Данная библиотека содержит большой набор общеупотребительных функции и классов. Полный интерфейс данной библиотеки - см. данную документацию.

Зависимости.

Для использования данной библиотеки (для ее сборки и работы с текущей версией) необходимы следующие библиотеки (зависимости от внешних библиотек):

  • log4j (1.2.15) - модуль ведения журнала работы/ошибок (логгирование).
  • commons-lang (2.4+) - модуль, содержащий большой набор общеупотребительных утилитарных функций.
  • commons-io (1.3.2+) - модуль, содержащий большой набор функций файлового ввода/вывода.
  • commons-cli (1.2+) - модуль для работы с командной строкой (cli - command line interface).
  • javamail (1.2) - модуль для работы с электронной почтой.
  • activation framework - данный модуль необходим для работы модуля javamail.
  • JUnit (4.8.2+) - фреймворк для тестирования исходного кода. Для сборки или использования библиотеки фреймворк не нужен.

Сборка проекта библиотеки.

Сборка данной библиотеки осуществляется с помощью сборочного скрипта ANT (используется версия 1.8.1). Описание сборочного скрипта ANT находится в виде комментариев в самом скрипте, описаны все сборочные задачи и порядок их следования, даны комментарии для сложных команд. Сборка может осуществляться как из среды IntelliJ IDEA 9.x.x и выше, так и напрямую из командной строки (без запуска среды IDE) с помощью утилиты ANT. Для сборки данной библиотеки необходимы все вышеуказанные библиотеки сторонних производителей (см. выше раздел "Зависимости"), кроме JUnit. Библиотека JUnit используется только для тестирования исходного кода классов библиотеки jlib, для работы собранной библиотеки jlib набор JUnit не нужен.

Использование логгирования.

Для работы системы логгирования следует инициализировать логгер с именем "jlib" - тогда все классы библиотеки смогут вести журналирование. Если же логгер инициализирован не будет, то при работе библиотеки будет возникать сообщение об этом (WARNING). Также следует инициализировать логгер с именем org.apache.commons для того, чтобы используемые библиотеки commons тоже могли вести журналирование.

Тестирование.

Начиная с версии библиотеки 4.9.0 исходный код библиотеки тестируется с помощью фреймворка JUnit 4.8.2. Для сборки и использования библиотеки модуля JUnit не нужны, они необходимы только для запусков тестов. Тест-кейсы (классы тестов) не включаются в итоговую сборку библиотеки. Документация javadoc не генерируется для тест-кейсов.
